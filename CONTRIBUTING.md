# Contributing to GTAF

Thank you for your interest in contributing to GTAF.

This document describes the **mechanics of contribution**.
It does not describe architectural rules, constraints, or design principles.

---

## 1. Required Reading

Before contributing, you must read:

- docs/README.md
- docs/STYLE.md
- docs/contributor-guide.md
- Relevant ADRs in docs/adr/

These documents define the rules and constraints of the system.

---

## 2. Contribution Workflow

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Update documentation when required
5. Submit a pull request

No special branch naming rules apply.

---

## 3. Pull Request Expectations

A pull request should:

- Clearly describe the change
- Reference relevant documentation or ADRs
- Be limited in scope

Reviews focus on correctness and clarity.

---

## 4. Documentation Changes

Documentation changes are welcome.

When modifying documentation:

- Follow the standard templates
- Keep changes scoped and explicit

---

## 5. Review Process

All pull requests are reviewed.

Reviewers may request:

- Documentation updates
- ADRs
- Scope reduction
- Clarification

---

## 6. Questions

If you are unsure about a change, open an issue or start a discussion
before submitting a pull request.
